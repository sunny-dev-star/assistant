# Agent Framework - 产品需求文档 (PRD)

## 文档信息
- **项目名称**: Agent Framework（智能体框架）
- **目标用户**: 中小企业（餐饮、教培、电商、律所等）
- **开发模式**: 个人开发者定制服务 + SaaS 多租户平台
- **文档版本**: v1.0
- **日期**: 2026-05-16

---

## 1. 项目背景与目标

### 1.1 背景
- AI 智能体（Agent）市场快速增长，但中小企业缺乏技术能力自行搭建
- 个人开发者需要一套可快速定制、低运维成本的智能体交付方案

### 1.2 目标
构建一套面向中小企业的智能体框架，具备：
- **纯自主引擎**: 基于 FastAPI + LiteLLM，完全掌控大模型路由、上下文管理与多租户隔离
- **快速定制**: 通过技能（Skill）机制适配不同行业需求
- **低运维成本**: 多租户 SaaS 架构，一次部署服务多个客户
- **可扩展**: 支持 MCP 协议接入外部工具
- **可监控**: 后台监控面板，实时掌握各客户智能体运行状态

---

## 2. 用户画像

### 2.1 主要用户：中小企业主/运营负责人
- **特征**: 不懂技术，有明确的业务痛点（获客、客服、内容）
- **需求**: 开箱即用，不需要维护，按效果付费
- **付费意愿**: 月预算 ¥1000-5000

### 2.2 次要用户：个人开发者（框架使用者）
- **特征**: 有技术能力，想快速交付 AI 项目给客户
- **需求**: 一套可复用的框架，减少重复开发
- **使用方式**: 基于框架开发行业技能，卖给终端客户

## 3. 功能需求（User Stories）

### 3.1 核心引擎层 (基于 FastAPI + LiteLLM)

#### US-001: 上下文管理
> **作为** 终端用户（企业客户），
> **我希望** 智能体能记住之前的对话内容，
> **以便** 获得连贯的交互体验。

**验收标准:**
- [ ] 支持多轮对话，AI 能引用之前的对话内容
- [ ] 上下文窗口可配置（默认 10 轮，最大 50 轮）
- [ ] 超长对话自动摘要，保留关键信息
- [ ] 支持会话隔离，不同客户的对话数据互不干扰

#### US-002: 多租户隔离
> **作为** 框架运营者（开发者），
> **我希望** 每个客户的数据完全隔离，
> **以便** 保障数据安全和隐私合规。

**验收标准:**
- [ ] 每个租户有独立的配置空间（LLM 模型、技能列表、知识库）
- [ ] 租户间数据物理隔离（数据库表前缀/独立 Schema）
- [ ] 租户资源配额管理（API 调用量、存储空间）
- [ ] 支持租户级别的开关控制（启用/停用/暂停）

#### US-003: LLM 接入与路由
> **作为** 框架运营者，
> **我希望** 支持多种 LLM 模型并自动路由，
> **以便** 根据成本和效果灵活选择。

**验收标准:**
- [ ] 支持 DeepSeek、通义千问、智谱、OpenAI 等主流模型
- [ ] 支持模型热切换，不需要重启服务
- [ ] 支持 Fallback 机制（主模型失败自动切换到备用模型）
- [ ] 支持按租户配置默认模型

#### US-004: 技能（Skill）加载机制
> **作为** 行业开发者，
> **我希望** 通过编写技能包来扩展智能体能力，
> **以便** 快速适配不同行业需求。

**验收标准:**
- [ ] 技能包采用标准化目录结构（见 4.1 节）
- [ ] 支持动态加载/卸载技能，不需要重启服务
- [ ] 技能间可以组合调用（一个技能可以调用另一个技能）
- [ ] 技能有独立的配置项和权限控制
- [ ] 提供技能开发 SDK 和示例模板

#### US-005: MCP 协议支持
> **作为** 框架运营者，
> **我希望** 智能体通过 MCP 协议连接外部工具，
> **以便** 扩展更多功能（查数据库、发邮件、调 API）。

**验收标准:**
- [ ] 支持 MCP Server 的自动发现与注册
- [ ] 支持 MCP Tool 的调用和结果回传
- [ ] MCP 连接状态监控（在线/离线/错误）
- [ ] 支持自定义 MCP Server 接入

#### US-006: 群组会话隔离与支持
> **作为** 框架运营者，
> **我希望** 框架原生支持多用户的群组会话，
> **以便** 智能体能在微信群、钉钉群等多用户场景下无缝工作。

**验收标准:**
- [ ] 架构底层原生支持 `user_id` 和 `name` 的绑定，能精准区分群组内的不同发言人
- [ ] 大模型上下文能准确识别不同用户的发言，避免出现“自问自答”等逻辑混乱
- [ ] 支持记录群组会话历史，且能在后台面板独立查看群聊记录

### 3.2 行业技能层（示例）

#### US-101: 餐饮 AI 客服技能
> **作为** 餐饮店老板，
> **我希望** AI 自动回复美团/大众点评/微信上的客户咨询，
> **以便** 减少人工客服成本，不漏掉任何一条消息。

**验收标准:**
- [ ] 支持接入美团商家版、大众点评、微信公众号
- [ ] 自动识别客户意图（订餐、咨询、投诉、评价回复）
- [ ] 根据菜单信息自动推荐菜品
- [ ] 支持预约订座功能
- [ ] 差评自动预警，推送给老板

#### US-102: 教培 AI 课程顾问技能
> **作为** 教培机构负责人，
> **我希望** AI 自动回答家长咨询并跟进意向学员，
> **以便** 提高转化率，减少销售人力。

**验收标准:**
- [ ] 支持课程信息问答（价格、时间、师资）
- [ ] 自动收集家长联系方式和意向信息
- [ ] 意向分级（高/中/低），自动推送给销售
- [ ] 支持试听预约
- [ ] 跟进提醒（3天未回复自动提醒销售）

#### US-103: 电商 AI 客服技能
> **作为** 电商卖家，
> **我希望** AI 7×24 小时回复买家咨询，
> **以便** 提升响应速度，减少差评。

**验收标准:**
- [ ] 支持淘宝、拼多多、抖音小店接入
- [ ] 自动查询订单状态、物流信息
- [ ] 处理退换货咨询
- [ ] 多语言回复（跨境场景）
- [ ] 复杂问题自动转人工

### 3.3 后台监控层

#### US-201: 实时监控面板
> **作为** 框架运营者，
> **我希望** 在后台看到所有客户智能体的运行状态，
> **以便** 及时发现和解决问题。

**验收标准:**
- [ ] 仪表盘展示：在线智能体数量、今日对话数、API 调用量
- [ ] 每个租户的实时状态（在线/离线/异常）
- [ ] 对话质量监控（响应时间、满意度评分）
- [ ] 异常告警（服务宕机、API 限额超限、错误率飙升）

#### US-202: 对话记录查看
> **作为** 框架运营者/客户管理员，
> **我希望** 查看智能体与用户的对话历史，
> **以便** 了解服务质量，优化话术。

**验收标准:**
- [ ] 按租户、时间、关键词筛选对话记录
- [ ] 对话内容脱敏展示（隐藏手机号等敏感信息）
- [ ] 支持标注"优质对话"和"问题对话"
- [ ] 导出对话记录为 Excel/CSV

#### US-203: API 用量统计
> **作为** 框架运营者，
> **我希望** 统计每个租户的 LLM API 调用量和成本，
> **以便** 控制成本，合理定价。

**验收标准:**
- [ ] 按租户统计 Token 消耗量
- [ ] 按模型统计调用成本
- [ ] 按技能统计调用频次
- [ ] 自动生成月度用量报告

#### US-204: 告警通知
> **作为** 框架运营者，
> **我希望** 系统异常时自动通知我，
> **以便** 及时响应，保障服务稳定性。

**验收标准:**
- [ ] 支持飞书/钉钉/企业微信/邮件告警
- [ ] 告警分级（紧急/重要/一般）
- [ ] 告警规则可配置（响应时间 > 5s、错误率 > 5% 等）
- [ ] 告警静默期设置（避免轰炸）

> **以便** 快速搭建新环境。

- [ ] 支持环境变量配置（API Key、域名等）

#### US-302: 多环境管理

> **我希望** 区分开发/测试/生产环境，
> **以便** 安全迭代。

**验收标准:**
- [ ] 支持 dev/staging/prod 三套环境配置

- [ ] 支持蓝绿部署/滚动更新

> **作为** 框架运营者，

**验收标准:**

## 5. 非功能需求

### 5.1 性能
- 单条消息响应时间 < 3 秒（95 分位）
- 系统支持 100 并发租户同时在线
- 单租户日对话量支持 10,000 条

### 5.2 安全
- 所有 API 通信使用 HTTPS
- 敏感数据（API Key、用户手机号）加密存储
- 支持 IP 白名单限制
- 对话内容脱敏，符合数据安全法规

### 5.3 可用性
- 系统可用性目标：99.5%（月度）
- 支持无中断更新（滚动部署）
- 关键组件有主备切换机制

### 5.4 扩展性
- 水平扩展：增加服务器即可提升容量
- 技能热插拔：新增技能不需要重启服务
- 模型热切换：更换 LLM 不需要重启服务

## 7. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| LLM API 成本不可控 | 高 | 设置用量上限，超限时自动降级到便宜模型 |
| 客户数据泄露 | 极高 | 数据加密、租户隔离、定期安全审计 |
| 技能开发进度慢 | 中 | 先聚焦 1-2 个行业，做深做透 |
| 客户流失率高 | 中 | 持续优化效果，提供数据报告证明价值 |

## 8. 附录

### 8.1 术语表
- **Skill（技能）**: 封装了特定行业能力的插件，包含提示词、工具、知识库
- **MCP**: Model Context Protocol，模型上下文协议，用于连接外部工具
- **Tenant（租户）**: 一个独立的客户实例，数据完全隔离
- **Token**: LLM API 的计费单位

### 8.2 参考文档
- MCP Protocol: https://modelcontextprotocol.io
- Dify Open Source: https://github.com/langgenius/dify
- OpenClaw Skills: https://docs.openclaw.ai
